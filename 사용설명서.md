## 1. 프로그램 소개

**AI 오피스**는 AI 에이전트 기반 업무 자동화 시스템입니다.

- 33명의 AI 에이전트가 11개 층 오피스에서 실제로 일합니다
- 업무 요청을 하면 적절한 부서에 자동 배분되고, 에이전트들이 협업해서 결과물을 만들어냅니다
- Claude, GPT-4o, Gemini 등 여러 LLM을 동시에 사용합니다

## 2. 화면 구성

### 사이드바 (왼쪽)

- **뷰 전환 버튼 7개**: Dashboard / AI Office / Tasks / Team Chat / Agents / Files / Settings
- **에이전트 목록**: 33명 전체가 부서별로 나열됨. 클릭하면 해당 에이전트 선택
- **접기/펼치기 버튼**: 사이드바를 아이콘만 보이게 축소 가능. 상태는 브라우저에 저장됨

### 헤더 (상단)

- **현재 뷰 이름** + 선택된 층 표시
- **빠른 제어판 버튼**: 층 이동(11개 버튼), 테마 전환, 뷰 전환
- **검색창**: 층 이름/에이전트 이름/메시지 내용으로 검색 → Enter로 해당 위치로 이동
- **알림 벨 (🔔)**: 최근 12개 항목(업무+메시지) 표시. 보면 읽음 처리됨
- **테마 토글**: 다크/라이트 모드 전환

## 3. 워크스페이스 뷰 7가지

### 3-1. Dashboard (대시보드)

**요약 카드 4개**
- 전체 에이전트 수
- 진행 중인 업무 수
- 전체 메시지 수
- 현재 층

**사업 지표 (Business KPI)**
- **업무 완료율**: 완료/(완료+실패) 비율을 막대 그래프로 표시
- **교차 검토율**: 다른 부서의 검토를 받은 업무 비율
- **부서별 처리량**: 상위 6개 부서의 완료/전체 업무 수 및 비율

**실행 로그 (Execution Log)**
- 최근 100개의 LLM 호출 / 도구 사용 / 메모리 검색 / 시스템 이벤트를 실시간으로 표시
- 에이전트가 무슨 작업을 하고 있는지 상세하게 볼 수 있음

**층별 현황 그리드**
- 11개 층 버튼에 각각 에이전트 수 / 활성 에이전트 수 / 메시지 수 표시
- 버튼 클릭 → 해당 층으로 이동

**최근 활동**
- 마지막 6개 메시지가 클릭 가능한 링크로 표시됨

### 3-2. AI Office (사무실)

**11개 층의 타일 기반 오피스**

| 층 | 부서 |
|---|---|
| 11F | 대표실 (CEO) |
| 10F | 전략·비서 (Executive) |
| 9F | 연구개발 (R&D 관리) · 경영지원·데이터 |
| 8F | 자동화개발 |
| 7F | 자동화개발 |
| 6F | 운영·오류대응 (QA + DevOps) |
| 5F | 제품기획 · 교육·서비스 |
| 4F | 세일즈 |
| 3F | 세일즈 |
| 2F | 마케팅·리서치 |
| 1F | 회의층 |

**에이전트 상태 표시**
- `idle` — 대기 중 (자리에 앉아 있음)
- `working` — 업무 처리 중
- `thinking` — 분석 중
- `debating` — 토론 중
- `moving` — 이동 중

**자율 행동 (Ambient Behavior)**
- 에이전트들은 유휴 시간에 자동으로 스트레칭, 산책, 카페 방문 등을 함
- 회의가 소집되면 1F 회의층으로 이동

**커뮤니케이션 패널 버튼**
- 화면 우측에 해당 층의 팀 채팅 패널을 열고 닫음

**카페 레이아웃 에디터 (1F 전용)**
- 1F에서 활성화 가능
- 가구 17종(테이블, 의자, 소파, 카운터, 선반 등)을 배치/회전/이동/삭제
- 커스텀 레이아웃은 저장되어 유지됨

---

### 3-3. Tasks (업무 관리)

**왼쪽 패널 — 업무 목록**
- 필터 버튼: `전체` / `대기` / `진행 중` / `승인 대기` / `완료` / `실패`
- 업무 클릭 → 오른쪽에 상세 내용 표시

**오른쪽 패널 — 업무 상세**
- 제목, 설명, 담당 부서 목록
- 현재 상태 (색상 배지)
- **승인 이유 표시**: 왜 승인이 필요한지 빨간색으로 표시 (예: "외부 커뮤니케이션", "가격 결정")
- **부서별 결과**: 각 팀이 기여한 내용 개별 표시
- **첨부 파일**: 업무에 포함된 파일 목록
- **승인/반려 버튼**: `승인 대기` 상태일 때만 활성화
- **재작업 입력창**: 반려 시 피드백 입력 가능
- **내보내기 버튼**: 이 업무 / 전체 업무를 마크다운으로 저장

---

### 3-4. Team Chat (팀 채팅)

**왼쪽 — 채널 목록**
- 부서별 채널이 목록으로 표시
- 읽지 않은 메시지 수 배지 표시
- **CEO 채널**: 전체 부서에 업무 요청 (라우팅 포함)
- **개별 부서 채널**: 해당 부서에만 직접 메시지

**오른쪽 — 메시지 창**
- 메시지 타입 배지: `업무` / `결과` / `토론` / `시스템`
- 파일 첨부 버튼 (📎)
- 메시지 입력 → Enter로 전송
- Shift+Enter로 줄 바꿈

---

### 3-5. Agents (에이전트 관리)

**왼쪽 — 에이전트 목록**
- 부서별로 그룹화된 33명
- 클릭하면 오른쪽에 상세 편집 창

**오른쪽 — 에이전트 편집**
- **이름**: 인라인 클릭 또는 폼으로 수정 가능
- **역할**: 수정 가능
- **LLM 모델 선택 드롭다운** (7가지):
  - Claude Opus 4.6
  - Claude Sonnet 4.6
  - Claude Haiku 4.5
  - GPT-4o
  - GPT-4o mini
  - Gemini 2.5 Pro
  - Gemini 2.5 Flash
- **추천 모델**: 해당 에이전트의 역할에 최적화된 모델과 이유 표시
- **상태 표시기**: 현재 에이전트 상태
- **저장 버튼**: 변경사항 저장 후 확인 피드백 표시

---

### 3-6. Files (파일 관리)

- 에이전트들이 생성한 출력 파일 목록
- 파일 아이콘, 이름, 크기, 수정 날짜 표시
- 클릭 → 내용 미리 보기 (코드는 모노스페이스 폰트)
- **새로 고침 버튼**: 최신 파일 목록 다시 불러오기
- **다운로드 버튼**: 선택한 파일 저장

---

### 3-7. Settings (설정)

4개 탭으로 구성됩니다.

#### 탭 1: 화면 (Appearance)
- **테마**: 다크 / 라이트 모드 선택
- **폰트 종류**: System / Serif / Mono / Pixel 중 선택
- **폰트 크기**: Small / Medium / Large 중 선택
- **응답 언어**: AI 에이전트 답변 언어 고정 (자동 / 한국어 / 영어 등)

#### 탭 2: AI 동작 (AI Behavior)

**메모리 시스템**
- 켜기/끄기 토글
- 저장된 메모리 목록 보기
- 전체 메모리 삭제

**승인 정책**
- `모든 업무 승인 필요` vs `정책 기반 자동 판단` 선택
- 5가지 승인 트리거 항목 개별 토글:
  - 외부 커뮤니케이션 (이메일, 공지)
  - 가격 결정 (견적서, 할인)
  - 결제 실행 (송금, 청구서)
  - 일정 확정 (미팅, 데모 예약)
  - 법적 약정 (계약서, NDA)

**자동 트리거 (Auto-Triggers)**
- 부서 간 연쇄 반응을 설정합니다
- 각 트리거 켜기/끄기 가능
- 메시지 템플릿 편집 가능

**지시사항 (Directives)**
- 현재 활성화된 회사 공지/회의 목록 표시
- 전체 삭제 버튼

#### 탭 3: 연동 (Integrations)

**웹훅 (Webhook)**
- 엔드포인트 URL 입력
- 이벤트 선택: 업무 완료 / 업무 실패
- 브라우저 알림 활성화 옵션

**Notion 연동**
- API 토큰 입력
- 데이터베이스 ID 입력
- 업무 완료/실패 시 Notion 페이지 자동 생성
- 속성: 상태, 담당자, 결과 포함

**스케줄러**
- 켜기/끄기 설정

**일일 토큰 예산**
- 켜기/끄기 토글
- 하루 사용 가능한 토큰 한도 설정
- 한도 초과 시 업무 실행 차단

#### 탭 4: 데이터 (Data)

**토큰 사용량**
- 제공자별 (Anthropic / OpenAI / Gemini) 사용량 표시

**백업 & 복원**
- 전체 데이터 내보내기 (JSON)
- 이전 백업 파일 업로드 → 복원
- 전체 데이터 초기화 (주의: 되돌릴 수 없음)

## 5. 업무 요청 방법

### 방법 1: CEO 채널 (전체 부서 협업)

1. **Team Chat** 뷰로 이동
2. 왼쪽 채널 목록에서 **CEO** 채널 선택
3. 메시지 입력 → Enter

이렇게 하면:
- 총괄 코디네이터가 어느 부서가 필요한지 LLM으로 자동 판단
- 해당 부서들에 업무 분배
- 각 부서 에이전트들이 협업
- CEO가 결과를 통합
- Tasks 뷰에서 진행 상황 확인 가능

### 방법 2: 부서 채널 (특정 부서에 직접)

1. Team Chat 뷰에서 원하는 부서 채널 선택 (예: 자동화개발, QA)
2. 메시지 입력 → Enter

해당 부서 에이전트들만 응답합니다.

### 파일 첨부

- 메시지 입력창 옆 📎 버튼 클릭
- 텍스트, 바이너리, ZIP 파일 지원
- ZIP의 경우 구조, 파일 수, 크기를 자동 분석하여 에이전트에게 컨텍스트로 전달

### 업무 진행 흐름

```
요청 입력
  ↓
부서 자동 배정 (CEO 채널) 또는 직접 배정
  ↓
각 부서 에이전트 순차 실행
  ↓
부서 내 결과 통합 (팀장급 에이전트가 취합)
  ↓
승인 필요 여부 판단
  ↓
[완료] 또는 [승인 대기 → 사용자가 승인/반려]
  ↓
웹훅 / Notion 연동 발동 (설정된 경우)
  ↓
자동 트리거 연쇄 실행 (설정된 경우)
```

### 긴 결과와 자동 이어쓰기

AI 제공사마다 한 번에 생성할 수 있는 출력량에는 자체 한도가 있습니다. AI 오피스는 긴 보고서나 표가 중간에서 끊기지 않도록, 출력 한도에 도달하면 자동으로 후속 요청을 보내 이전 결과 뒤에 이어 붙입니다.

- 일반 사용자는 별도로 조작할 필요가 없습니다.
- 결과가 길면 한 번의 응답 안에서 여러 차례 자동 이어쓰기가 실행될 수 있습니다.
- Claude 계열 모델은 안정성을 위해 서버에서 안전한 출력 한도로 조정하고, 필요한 경우 자동으로 이어 씁니다.
- GPT/Gemini 계열은 더 긴 출력 한도를 사용할 수 있으며, 마찬가지로 중간 중단 시 자동 이어쓰기를 시도합니다.
- 자동 이어쓰기를 여러 번 시도해도 제공사 한도에 계속 도달하면 결과 끝에 `[출력 중단 안내]`가 표시됩니다.
- 이 안내가 보이면 같은 스레드에서 `이어서 계속`이라고 입력하면 남은 내용을 이어서 받을 수 있습니다.

`500 Internal Server Error`는 보통 일일 토큰 예산 초과가 아니라 서버 또는 LLM 제공사 호출 오류입니다. 일일 토큰 예산을 초과한 경우에는 설정된 예산 정책에 따라 별도 차단/경고가 표시됩니다.

## 6. 토론 기능

두 부서의 AI 에이전트들이 특정 주제를 놓고 실제로 토론하게 할 수 있습니다.

### 사용법

CEO 채널 또는 아무 채널에서 메시지 앞에 `@토론`을 붙여 전송합니다.

```
@토론 새 인증 방식을 OAuth로 할지 자체 개발할지
@토론 보안 강화 vs 개발 속도 중 어느 쪽이 우선인가
```

### 진행 흐름

```
@토론 [주제] 입력
  ↓
키워드로 관련 부서 2개 자동 선정
  ↓
두 부서 동시에 [초기 의견] 수집
  (각 팀원이 역할별로 분담해 개별 의견 작성 → 팀장이 자동 조합)
  ↓
두 부서 동시에 [반론] 수집
  (상대 부서 의견을 읽고 반박 정리)
  ↓
CEO가 최종 입장 요약 제시
  ↓
사용자의 최종 결정 대기
```

### 특징

- **부서 자동 선정**: 주제 키워드를 분석해 가장 관련 있는 두 부서가 자동으로 참여
- **역할 분업**: 부서 내 에이전트들이 각자 전문 영역을 나눠 의견을 작성하고 팀장이 합산
- **2라운드 구조**: 초기 의견 → 반론 순서로 진행. 두 부서가 병렬로 동시 처리
- **에이전트 상태**: 토론 중인 에이전트는 오피스에서 `debating` 상태로 표시됨
- **결과 채널**: 관련 부서 채널에 `토론` 배지가 붙은 메시지로 기록됨

### 주의사항

- 부서 자동 선정이므로 원하는 부서를 직접 지정하려면 키워드를 명시적으로 포함할 것
- 토론은 업무(Task)로 등록되지 않으며, 팀 채팅 메시지로만 기록됨
- 최종 결정은 사용자가 직접 내려야 함. 에이전트들은 결정하지 않음

## 7. 승인 시스템

### 승인이 필요한 경우

5가지 정책 중 해당되는 내용이 업무 결과에 포함되면 자동으로 **승인 대기** 상태가 됩니다:

- 외부 이메일 발송, 고객 공지 등
- 가격 견적, 할인 제안 등
- 송금, 청구서 발행 등
- 미팅 일정 확정, 데모 예약 등
- 계약서, NDA 서명 등

### 승인 처리 방법

1. Tasks 뷰에서 `승인 대기` 필터 클릭
2. 해당 업무 선택
3. 승인 이유 확인 (빨간색 텍스트로 표시)
4. **승인 버튼**: 업무 완료 처리 → 웹훅/Notion 발동
5. **반려 버튼**: 피드백 입력창 나타남 → 이유 입력 후 반려

## 8. 메모리 시스템

에이전트들은 이전 업무 내용을 기억합니다.

### 작동 방식

- 업무 완료 시 자동으로 **제목, 요약, 핵심 포인트(5개), 태그(6개), 관련 부서** 추출하여 저장
- 최대 200개까지 저장
- 새 업무 시작 시 관련된 메모리를 자동 검색 (최대 3개)
  - 의미적 유사도(임베딩) 기반 검색
  - 키워드 기반 폴백 검색
  - 같은 부서의 메모리에 가중치 부여

### 관리

- Settings → AI 동작 탭 → 메모리 섹션
- 전체 메모리 목록 보기
- 특정 메모리 삭제 / 전체 삭제

## 12. 지시사항(Directives) 시스템

지시사항은 에이전트들의 행동에 영향을 주는 **전사 공지 및 회의 소집** 기능입니다.  
CEO 채널 또는 부서 채널 입력창에 자연어로 입력하면 자동으로 감지됩니다.

---

### 12-1. 전사 공지 (Announcement)

아래 접두어로 시작하거나 포함하면 전사 공지로 등록됩니다.

```
전사 공지: 보안 점검 기간 동안 외부 배포 금지
전직원 공지: 이번 주 금요일 전체 회의 예정
전체 공지: 고객 데이터 취급 시 반드시 암호화 적용
전사 지침: 신규 계약건은 법무 검토 후 진행
```

- 모든 부서 에이전트에게 즉시 적용됨
- 이후 업무, 토론, 자율 행동에 지속적으로 반영
- 새로운 전사 공지가 등록되면 기존 공지는 자동으로 교체됨

**공지 해제:**
```
공지 해제
전사 공지 해제
지침 해제
```

---

### 12-2. 회의 소집 (Meeting)

| 규모 | 참여 부서 수 | 장소 | 상태 표시 |
|---|---|---|---|
| 대회의 | 전부서 | 1F 대회의실 | `대회의실 집결 중` |
| 중회의 | 핵심 부서 | 1F 중회의실 | `중회의실 집결 중` |
| 소회의 | 1~3개 부서 | 1F 소회의실 | `소회의실 협의 중` |

**소집 방법:** 아래 키워드 중 하나 + 지시 어조(`하세요`, `합시다`, `해주세요` 등)를 함께 입력

```
전 팀 회의실로 모이세요
보안팀과 개발팀 소회의실로 집결해주세요
전 직원 대회의실 소집합니다
중회의 시작합시다
```

- 소집된 에이전트들은 `moving` 상태로 1F 회의실 좌석에 배치됨
- 회의 중 업무 요청이 오면 해당 지시가 해제될 때까지 작업이 중단됨

**해산 방법:** 아래 중 하나를 입력

```
회의 종료
회의 해산
해산하세요
복귀하세요
원위치 복귀
집합 해제
회의를 마쳤습니다
회의 마무리
```

- 에이전트들이 출구 방향으로 이동 후 원래 층/자리로 자동 복귀
- 복귀 애니메이션이 실행되고 `idle` 상태로 전환됨

---

### 12-3. 관리

- **Settings → AI 동작 탭 → 지시사항 섹션**: 현재 활성 공지/회의 목록 확인 및 삭제
- 공지와 회의 소집은 동시에 각각 1개씩 유지됨 (새로 등록하면 기존 것 교체)
