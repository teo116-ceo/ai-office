# AI 오피스 - 지음과깃듬 휴먼 데이터 기업 시뮬레이터

## 프로젝트 개요
픽셀 아트 기반의 자율형 AI 에이전트 오피스. 주식회사 지음과깃듬의 휴먼 인사이트 데이터 사업을 AI 에이전트 조직으로 구현한다.

## 회사 정의
지음과깃듬은 단순 진단 서비스 회사가 아니라 휴먼 데이터 생산 기업이다.

핵심 흐름:
```
진단 도구
→ 데이터 생성
→ 해석 / 리포트
→ 의사결정 지원
```

## 사업 구조
- 개인 기질 진단 기반 리포트 서비스
- 전문 강사 양성 및 진단 도구 판매
- 기관 연간 진단 라이선스
- 조직 진단 연구개발
- 창업자 진단 베타 및 VC/AC·정부기관 대상 확장
- 리포트 자동화 및 사업 결과 보고용 통계 데이터 고도화

## 기술 스택
- Vite + React + TypeScript
- Tailwind CSS
- Zustand (전역 상태 관리)
- Codex API (Anthropic SDK)
- pixel-agents (픽셀 캐릭터 렌더링)

## 폴더 구조
```
src/
├── components/
│   ├── layout/       # Sidebar, Header, CommunicationPanel
│   ├── office/       # OfficeCanvas, AgentCharacter, Room
│   └── ui/           # 공통 UI 컴포넌트
├── store/            # Zustand 스토어
├── services/         # Codex API 연동
├── types/            # TypeScript 타입 정의
└── assets/sprites/   # 픽셀 아트 에셋
```

## 에이전트 조직 구성
- 대표/총괄: 업무 분배, 우선순위 결정, 최종 통합
- 전략/비서: 대표 일정, 우선순위, 사업 포트폴리오 정리
- R&D 관리: 진단 주제와 개발 단계 관리
- 데이터 관리: 진단 결과 축적, 패턴 분석, 리포트 자동화
- 경영지원: 회계, 비용/매출, 세무 대응 데이터 정리
- 자동화개발: 리포트 자동화, 데이터 파이프라인, 내부 도구 구현
- 오류대응: 진단 오류 접수, 분류, 해결 트래킹
- 운영자동화: 라이선스, 배포, 백업, 운영 프로세스 자동화
- 제품기획: 개인/기관/조직/창업자 진단 제품화
- 교육운영: 강사, 강의, 자격증, 시험 운영
- 세일즈: 리드, 파이프라인, 계약 상태 관리
- 리서치: HR, 창업, AI, 경제/시사 트렌드 수집
- 마케팅: 콘텐츠, 캠페인, 타겟 분석

## 코딩 컨벤션
- 컴포넌트: PascalCase (예: AgentCharacter.tsx)
- 함수/변수: camelCase
- 타입/인터페이스: PascalCase + 접두사 T or I 없이 (예: Agent, Department)
- 상수: UPPER_SNAKE_CASE
- CSS: Tailwind 클래스 우선, 커스텀 CSS 최소화

## Do Not
- 사용자 허락 없이 기존 파일 삭제 또는 덮어쓰기 금지
- 모르는 에러 임의 추측 금지, 모른다고 먼저 말할 것
- Codex API 키를 코드에 하드코딩 금지 (.env 사용)
- 한 컴포넌트에 300줄 이상 작성 금지 (분리할 것)
- any 타입 사용 금지 (unknown 또는 명시적 타입 사용)
